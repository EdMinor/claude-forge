### Фаза 0 — MVP
**Цель:** Проверить core гипотезу с минимумом кода
**Срок:** [2-4 недели]
**Принцип:** Только то без чего продукт не работает вообще

#### Включено в MVP
- [ ] Авторизация (email+password)
- [ ] [Core Feature 1] — [что именно, не абстрактно]
- [ ] [Core Feature 2]
- [ ] [Core Feature 3]
- [ ] Базовый UI (функциональный, не финальный)

#### НЕ включено в MVP (сознательно)
- [ ] [Feature X] → Фаза 1
- [ ] [Feature Y] → Фаза 2
- [ ] [Feature Z] → После релиза

#### Критерии готовности MVP
```
[ ] Пользователь может зарегистрироваться
[ ] Пользователь может выполнить core action (главное действие продукта)
[ ] Данные сохраняются и не теряются
[ ] Работает на мобильном браузере
[ ] Нет критических ошибок в happy path
```

#### Метрика успеха MVP
[Что конкретно должно произойти чтобы считать MVP успешным]
Пример: "10 пользователей вернулись на второй день" или "первая платная конверсия"
